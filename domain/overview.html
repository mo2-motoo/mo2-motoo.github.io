🎯 도메인 설명 개요

Motoo 프로젝트의 핵심 비즈니스 도메인과 개념

📋 도메인 개요

Motoo는 주식 거래 플랫폼으로, 여러 핵심 비즈니스 도메인으로 구성되어 있습니다. 각 도메인은 독립적인 책임을 가지며, 서로 협력하여 전체 시스템을 구성합니다.

🎯 핵심 도메인

  • 사용자 관리: 회원가입, 로그인, 프로필 관리
  • 종목 관리: 주식 정보, 시세 데이터, 종목 검색
  • 주문 처리: 매수/매도 주문, 주문 상태 관리
  • 포트폴리오: 보유 주식, 수익률 계산, 자산 관리
  • 거래내역: 거래 기록, 내역 조회, 정산
  • 토큰 관리: 인증 토큰, 권한 관리, 보안
  • 체결 처리: 주문 체결, 매칭 엔진, 실시간 처리

🏗️ 도메인 설계 원칙

단일 책임 원칙

각 도메인은 하나의 명확한 책임만을 가집니다.

느슨한 결합

도메인 간 의존성을 최소화하여 독립성을 보장합니다.

데이터 무결성

각 도메인은 자신의 데이터 무결성을 보장합니다.

일관성 보장

도메인 간 데이터 일관성을 유지합니다.

🔄 도메인 간 통신

동기 통신

  • 사용자 → 토큰: 인증 요청
  • 주문 → 종목: 종목 정보 조회
  • 포트폴리오 → 거래내역: 거래 내역 조회

비동기 통신

  • 주문 → 체결: 주문 체결 요청
  • 체결 → 포트폴리오: 보유 주식 업데이트
  • 체결 → 거래내역: 거래 기록 생성

이벤트 기반 통신

  • 주문 체결 이벤트: 여러 도메인에 알림
  • 시세 변동 이벤트: 포트폴리오 가치 업데이트
  • 사용자 활동 이벤트: 로그 및 분석
